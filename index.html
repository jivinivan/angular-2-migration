Preparing for Angular 2

Remove $scope, use controllerAs

Why remove $scope?

  • There is no $scope in Angular 2
  • It's best practice to use controllerAs in your Angular 1.x apps so you should do it anyway

TypeScript

Microsoft language (superset of JavaScript)

Ng2.0 natively written in TS

  • Static typing
  • IDE code navigation and autocomplete - better tooling
  • Less unit tests needed?
  • Low risk to implement
  • ... but... yet another THING
  • ES6/Babel

Move to component based directives

Why use a directive?

  • Reusability
  • Everything is a component in Angular 2
  • module.component

Put your business logic in a Service

This is great preparation for Angular 2, and also best practice for Angular 1.x

But why? Why put my business logic in a Service and not in a Controller?

  • Typical use of a Service (read/write to api - persistence)
  • Common misconception - only use services for api interaction. NOT SO!!!
  • Services are singletons, yo
  • Encapsulate your business logic in a singleton so that it can be shared throughout your app

Services are really easy to migrate to Angular 2

  • How easy? Copy and paste if they are written as basic singleton classes.
